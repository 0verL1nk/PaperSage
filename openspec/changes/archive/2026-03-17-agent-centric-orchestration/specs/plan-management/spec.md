## ADDED Requirements

### Requirement: Plan Creation by Leader
The system SHALL provide a create_plan tool that stores Leader's self-written plan.

#### Scenario: Leader creates plan with own content
- **WHEN** Leader calls create_plan with goal and description
- **THEN** the system stores the plan in agent state without generating content
- **AND** the plan content is exactly what Leader provided

#### Scenario: Plan is storage only
- **WHEN** create_plan is invoked
- **THEN** the tool does NOT call LLM to generate plan content
- **AND** the tool does NOT use planning_service.build_execution_plan

#### Scenario: Plan already exists
- **WHEN** Leader calls create_plan while a plan already exists
- **THEN** the system replaces the existing plan with the new one
- **AND** the system indicates that the previous plan was replaced

### Requirement: Plan Reading
The system SHALL provide a read_plan tool that retrieves Leader's plan.

#### Scenario: Read existing plan
- **WHEN** Leader calls read_plan and a plan exists
- **THEN** the system returns plan_id, goal, and description as written by Leader

#### Scenario: No plan exists
- **WHEN** Leader calls read_plan and no plan exists
- **THEN** the system returns a message indicating no active plan

### Requirement: Plan Update by Leader
The system SHALL provide an update_plan tool that modifies Leader's plan description.

#### Scenario: Leader updates plan description
- **WHEN** Leader calls update_plan with new description
- **THEN** the system updates the plan's description field with Leader's new content
- **AND** the tool does NOT regenerate or modify Leader's content

#### Scenario: Update non-existent plan
- **WHEN** Leader calls update_plan and no plan exists
- **THEN** the system returns an error indicating no plan to update

### Requirement: Plan Deletion
The system SHALL provide a delete_plan tool that removes the current plan.

#### Scenario: Delete existing plan
- **WHEN** agent calls delete_plan and a plan exists
- **THEN** the system removes the plan from agent state

#### Scenario: Delete non-existent plan
- **WHEN** agent calls delete_plan and no plan exists
- **THEN** the system returns a message indicating no plan to delete

### Requirement: Plan Storage
Plan data SHALL be stored in agent state and persisted via checkpointer.

#### Scenario: Plan persistence via checkpointer
- **WHEN** a plan is created
- **THEN** it is stored in the agent's state
- **AND** SqliteSaver checkpointer automatically persists it to database

#### Scenario: Cross-session access
- **WHEN** user returns to the same session (thread_id)
- **THEN** the plan is restored from checkpointer
- **AND** user can continue working with the same plan

### Requirement: Tool Visibility
Plan management tools SHALL be marked as lazy for progressive disclosure.

#### Scenario: Initial tool list
- **WHEN** Leader starts a new conversation
- **THEN** plan management tools are not visible in the initial tool list

#### Scenario: Tool activation
- **WHEN** Leader calls search_tools for "plan"
- **THEN** plan management tools become visible

### Requirement: No System-Generated Plans
The system SHALL NOT use planning_service to generate plans.

#### Scenario: No build_execution_plan calls
- **WHEN** plan tools are implemented
- **THEN** they do NOT call planning_service.build_execution_plan
- **AND** they do NOT use any LLM to generate plan content

#### Scenario: Leader is the author
- **WHEN** a plan exists in agent state
- **THEN** its content was written by Leader, not generated by system
